# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, share the details privately with the maintainer and include:

- a description of the issue
- affected versions or commit range, if known
- reproduction steps or a proof of concept
- the potential impact
- any suggested remediation, if available

## Scope

Security reports are especially helpful for issues involving:

- command execution
- path handling
- environment variable handling
- dependency or packaging risks
- logging of sensitive data

## Response

This repository is still early-stage, so response times may vary. Good-faith reports will be reviewed and handled as quickly as possible.
